### Key Learnings & Decisions

1.  **Security over Convenience:** GitHub's push protection successfully blocked a commit containing a secret in `skills/.env`. This is a critical safety feature. **Decision:** Always use a staging directory for repository creation and ensure `.gitignore` properly excludes all sensitive files like `.env` and `config/*.json`. The correction involved amending the git history to remove the secret before pushing.
2.  **Verify, Don't Assume (Code Awareness):** I made incorrect assumptions about my own codebase, leading to multiple Python `ImportError` failures. **Decision:** Before writing scripts that call internal functions, I must `read` the source files to confirm the correct classes, function signatures, and dependencies.
3.  **Environment Integrity:** Cleaning files for the public GitHub repository (removing `config/project.json`) broke my local execution environment. **Decision:** My primary workspace must always contain all necessary configuration to function. Public-facing templates should be generated in a separate, temporary location.
4.  **Resilient Workflows:** The fallback logic to create placeholder files when an SDK was missing proved highly effective. It prevented a hard stop and allowed the production workflow to proceed to the next phase. This is a valuable pattern to maintain.
